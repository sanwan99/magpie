Prototype Spec · v0.1

Magpie

一个本地优先、键盘驱动的 macOS 剪切板管理器。 像喜鹊一样，把你随手复制的零碎都收集起来——文本、代码、链接、图片、文件、文件夹路径—— 用一次按键召唤，一次按键粘贴。

形态 屏幕底部条形面板（Dock 式） 平台 macOS 14+ 建议技术 SwiftUI / Tauri / Electron + React 原型文件 Deck Clipboard.html

01产品定位

先定调，再做事。决策一旦定下，所有后续设计都围绕它收敛。

三句话原则

  1. 本地优先 · 隐私第一。所有数据存本地 SQLite，可选 Touch ID 解锁；不联网、不上传、不分析。
  2. 键盘是一等公民。从呼出到粘贴，全流程不需要触碰鼠标。每个鼠标可达的功能都必须有快捷键。
  3. 类型即视觉。不同类型（文本/代码/链接/图片/文件/文件夹）有专门的预览，第一眼就知道是什么。

不做什么

  • 不做云同步、账号系统、协同编辑（保持本地纯净）。
  • 不做花哨配色或品牌渐变。色彩极简（黑白灰），强调"内容本身"。
  • 不做 AI 摘要 / 推荐（避免噪音；后续如需加入须另开抽屉）。

名字与隐喻

Magpie（喜鹊）——天生爱收集闪亮小物件的鸟。这隐喻贯穿始终： 复制即"叼回来"、pin 即"塞进巢里"、queue mode 即"按顺序往外掏"。 Logo 建议是一个简洁的喜鹊侧影或衔着方形光斑的极简符号。

03数据模型

每条剪切记录是一个 Clip。共 6 种类型，结构相似但 type-specific 字段不同。

Clip（统一字段）

// 所有 Clip 都有这些 { id: "c01" // uuid / nanoid type: ClipType // 见下 app: "vscode" // 来源 app 标识 createdAt: Date // 复制时刻 pinned: Bool // 是否置顶 tags: ["react", "snippet"] title: "useDebounce hook" // 可选；自动生成或用户改 }

ClipType — 6 种

type专属字段用途预览要点
text body: String 纯文本/段落 前 6 行截断，等宽 + 普通字体混排
code body: String
lang: String		 代码片段，带语言提示 语法高亮，detail 视图带行号
url url, host, title, desc 链接（可拉 OGP 信息） favicon + 标题 + 摘要 + 缩略图
image w, h, sizeKB, blob 截图/位图 缩略图 + 尺寸 overlay
file path, kind, sizeKB 单个文件路径 文件图标 + 名 + 路径（mono）
folder path, items 用户主用：复制文件夹路径 文件夹图标 + 名 + 路径 + 内含文件数

Snippet（用户定义模板）

{ id: "s01" shortcut: ";sig" // 触发字符串，输入时自动展开 title: "Email signature" body: "— Yu\nProduct & Design" }
实现提示：底层只用一张表 clips，所有类型走 JSON 字段。 Snippets 单独一张表 snippets。 搜索建立 FTS5 虚拟表（SQLite 全文检索）覆盖 title / body / url / host / desc / path / tags。

04界面与组件

面板由 4 个区域 + 5 个组件级元素组成。

4 个区域

① Panel Head

搜索框（带计数器）、类型筛选条、Pin/Queue 开关、布局切换、Snippets / Settings 入口。所有过滤条件都在这一行。

② Panel Body

左侧：当前布局（Stripe / Stack / Grid）展示 clips。右侧：Detail Pane（Stack 模式下更宽）。

③ Panel Foot

状态条：当前选中的类型 / 来源 app / 复制时间；右侧滚动展示主要快捷键。Queue Mode 开启时显示 Pill。

④ Floating Layers

Snippets Rail（面板上方）、Settings Sheet（居中）、Paste Toast（粘贴反馈）、Dock Hint（面板隐藏时的呼出提示）。

5 个核心组件

组件职责关键 props
<ClipCard> 列表中的单条目卡片，类型敏感的预览 clip, focused, dense
<ClipPreview> 类型分发的紧凑预览（switch on type） clip, dense
<ClipDetail> 右侧详情面板：标题 + 完整预览 + 操作按钮 clip, onPin, onPaste
<CodeBlock> 代码渲染（轻量语法着色，可选行号） src, lang, lineNumbers, maxLines
<FilterRail> 顶栏类型筛选条（All / Text / Code / Link / Image / File / Folder） value, onChange

05交互行为

每一个动作都要"短"——最多两步完成。

呼出与隐藏

  • 呼出：全局 ⌘+P 触发。面板从屏幕底部滑入（300ms ease-out），同时自动 focus 搜索框。
  • 隐藏：Esc 在搜索为空且无浮层时，下滑退出。或再次按 ⌘+P。
  • 失焦自动隐藏（设置可关）：当面板失去 key window 状态超过 200ms，自动收起。

选择与导航

  • ←/→ 或 ↑/↓ 切换 focus（焦点项滚动入视）。
  • 鼠标单击：选中。鼠标双击：直接粘贴。
  • Space：切换右侧 Detail Pane 显隐。

粘贴

  • ↵：粘贴当前 focus 项。先关闭面板，再写入系统剪切板，再向前一个 active app 模拟 ⌘+V。
  • ⇧+↵：粘贴为纯文本（剥离格式）。
  • ⌘+1…9：跳过选择，直接粘贴第 N 项。
  • 粘贴后中央显示 toast "Pasted"（700ms 淡出）。

Pin / 搜藏

  • ⌘+D：toggle 当前 focus 项的 pinned。Pin 项始终排在过滤结果最前。
  • 右上角"Pinned"开关：仅显示 pin 项。

Queue Mode（队列粘贴）

  • ⌘+Q：开启 / 关闭。底栏显示 Pill 指示器。
  • 开启后，每次 ↵ 粘贴当前项，然后 focus 自动前进一项。适合连续粘贴多条到表格、表单。
  • 关闭时回到普通模式。

Folder 类型的特殊行为

用户的核心场景：复制文件夹路径。 ↵ 默认粘贴 path 字符串（不是 Finder 引用）。 ⇧+↵ 粘贴 POSIX 绝对路径（展开 ~）。 ⌥+↵ 粘贴 file:// URL。 详情面板有"Press ↵ to copy path"提示，避免歧义。

07搜索与过滤

普通用户只需打字；高级用户用 key:value 语法。

语法

  • 关键词：多个空格分隔的词为 AND 关系，命中 title / body / url / host / desc / path / tags。
  • 类型过滤：type:code · type:folder · type:url
  • 来源 app：app:vscode · app:terminal
  • 标签：tag:design
  • 组合：type:code react hook = 类型为 code，且 body/title 含 react 和 hook。

UI 行为

  • 实时过滤（无 debounce 太刻意；用 16ms 节流够了）。
  • 搜索框右侧实时显示结果计数。
  • 顶部类型筛选条与搜索语法 合取：两者都要满足。
  • 当结果为空：展示居中"No matches"占位 + "Esc 清除筛选"提示。

08三种布局

同一份数据，三种节奏。用户可按场景切换；偏好持久化。

Stripe — 横向条形流（默认）

  • 水平滚动卡片，每张固定宽 220px、高 220px，间距 10px。
  • 最贴合"屏幕底部 Dock"形态。视觉密度中等。
  • focus 项轻微上移 2px + 投影增强。
  • 右侧固定一个 360px 的 Detail Pane。

Stack — 居中列表

  • 垂直一行一个，每行 4 列：⌘N | type icon | title + meta | mini preview。
  • 信息密度最高，适合深度检索。
  • 右侧 Detail Pane 更宽（380px）。

Grid — 紧凑网格

  • repeat(auto-fill, minmax(170px, 1fr))，瓦片高 170px。
  • 适合视觉内容（图片、颜色——虽然我们移除了颜色——、文件夹）。
  • Detail Pane 可关。
建议：默认 Stripe；让用户自然发现 Stack 和 Grid，但不强推。 布局偏好存 UserDefaults，跟随 app 启动恢复。

09设置面板

居中模态，macOS 标题栏风格（左上角红黄绿小圆点）。

4 个分页

General

  • Theme：Light / Dark（也可后续加 Auto follow system）。
  • Vibrancy：毛玻璃强度滑块（0–60px）。
  • Accent：Mono / Graphite / Blue / Olive 四档低饱和强调色。
  • Launch at login · Show recent first · Detect colors and links · Strip tracking parameters from URLs（开关组）。

Shortcuts

  • 所有快捷键的对照表。每行右侧渲染 标记。
  • v0.1 仅展示；v0.2 允许重绑（点击行进入捕获模式）。

History

  • Keep history for：7 / 30 / 90 days / Forever。
  • Max items：默认 2000，超出按时间淘汰非 pin 项。
  • Ignore apps：黑名单（不捕获 1Password、Keychain Access 等）。
  • Clear history：危险按钮，需二次确认。

Privacy

  • Touch ID to unlock：每次启动需要 Touch ID。
  • Encrypt local store：用 Keychain 存 SQLCipher 密钥。
  • Skip secret-looking content：正则识别 API key / token / OTP，不入库。
  • Send analytics：默认 OFF，永不开。

10Snippets（用户模板）

面板上方抽屉。打字 ;sig 或在抽屉里点选即可粘贴对应模板。

  • 每条 Snippet 有 shortcut、title、body 三字段。
  • 点击抽屉中的条目：把 shortcut 写进搜索框，或直接粘贴（设置可选）。
  • 底部"+ New snippet"进入编辑器（v0.2）。
  • 建议预置：;sig（签名）;meet（会议模板）;date（ISO 日期）;review（PR 检查表）。

极简黑白灰；vibrancy 是主角，色彩永远是配角。

排版

  • 主字体：SF Pro Text（系统字体），13px / 1.45。
  • 等宽：SF Mono，用于代码、路径、hex 等。
  • 标题（详情面板）：16px / 600 weight。
  • 所有大写小标签：11px / 600 / letter-spacing 0.06em。

圆角与阴影

  • 面板 18px，卡片 12px，瓦片 10px，按钮 7-8px。
  • 阴影分层：1px inset 高光 + 24px 远阴影 + 4px 近阴影。Dark 下加重 50%。

毛玻璃

  • backdrop-filter: blur(36px) saturate(170%) 是默认。
  • 下层必须有内容（桌面壁纸、其他窗口），否则毛玻璃失去意义——这点对 macOS 原生实现来说自动满足。

动效

  • 面板呼出：cubic-bezier(.2,.7,.2,1)，350ms。
  • focus 切换：scroll-behavior: smooth，让焦点条目滑入视野。
  • 粘贴 toast：进 250ms，停 400ms，出 300ms。
  • 所有 hover：120ms 线性。

12技术建议

本地使用、个人项目。建议用最省心的栈。

方案 A · SwiftUI（最贴近原型）

  • 原生体验、完美 vibrancy（NSVisualEffectView）、Touch ID 一行 API。
  • 剪切板监听：NSPasteboard.general.changeCount 轮询（400ms）。
  • 全局快捷键：HotKey Swift package。
  • 持久化：SQLite via GRDB；Snippet/Tag 简单 JOIN。
  • 粘贴回前台：保存 NSWorkspace.frontmostApplication，关闭面板后激活并模拟 ⌘V。

方案 B · Tauri + React（跨平台空间，单文件可分发）

  • 前端就是当前 HTML/JSX 原型，几乎可以直接搬。
  • Rust 端做剪切板监听（arboard crate）+ 全局快捷键（tauri-plugin-global-shortcut）。
  • 窗口设为 always-on-top + transparent + 不抢焦点。
  • 体积比 Electron 小得多（~10MB）。

方案 C · Electron + React（最简单）

  • 原型代码几乎零改动。
  • 缺点：体积大（120MB+）、毛玻璃不完美。
  • 仅推荐快速 MVP。

共通要点

  • 剪切板写入：粘贴前先备份当前 pasteboard，粘贴后 200ms 还原（可选；避免污染用户原本要复制的内容）。
  • 图片存储：>1MB 的图存为单独文件，DB 只存路径。
  • 应用归因：监听到剪切板变化时立刻记录 NSWorkspace.frontmostApplication.bundleIdentifier。
  • 类型识别：先按 pasteboard type，再按内容嗅探（folder = 目录路径且 isDirectory 为真）。

13实现路线
